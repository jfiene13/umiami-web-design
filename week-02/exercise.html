Exercise: Web Graphics

To do this exercise, find an image released into the public domain. Many government entities (U.S. Military, NASA, National Parks Service, etc.) provide such photos. Download a high-resolution image (preferably 300dpi). Remember to document where you downloaded the image from so you can cite and link back to the source.

Instructions

  1. Create a directory, web-graphics.
  2. Within web-graphics, create another directory, img.
  3. Re-size your image to about 400 pixels by 300 pixels (or flipped if your photo has a vertical orientation).
  4. Save your images in the img directory in the following formats:
    • GIF
    • PNG-8
    • PNG-24
    • JPEG, Low quality
    • JPEG, Medium quality
    • JPEG, High quality
    • JPEG, Very high quality
  5. When you name your images, come up with a standarn pattern for each. See the week-02/web-graphics/img folder on GitHub for examples.
  6. Record what the file size (in kilobytes) is for each image.

In a plain text file, write about 300-400 words about each file format (GIF, PNG and JPEG) and its benefits. Would you use one format over another? How do the different images differ in quality and size? Can you see where there is a good tradeoff between size and quality in the different JPEG files? What about the PNG and GIF files?

Your explanation should be well-written, but as if you were explaining this to a friend who has asked you for advice about their own web site. (And don't copy and answer the questions I've listed above.)

Save your web-graphics directory and its contents to your GitHub account.
